## 2026-05-21 - Fix Path Traversal in FileResponse catch-all route
**Vulnerability:** A catch-all route in FastAPI using `FileResponse(dist_dir / full_path)` allowed Local File Inclusion / Path Traversal via raw socket connections because `FileResponse` lacks built-in bounds checking (unlike `StaticFiles`), allowing requests like `GET /../server/main.py HTTP/1.1` to access arbitrary files.
**Learning:** `FileResponse` must always be paired with explicit `.resolve().is_relative_to(base_dir.resolve())` bounds checking if serving user-provided paths. FastAPI's `TestClient` and HTTP libraries (like `httpx` or `requests`) normalize paths, hiding the vulnerability during normal testing; it requires raw socket testing to uncover.
**Prevention:** Always validate that resolved file paths remain within their intended directory boundaries before passing them to `FileResponse`. Use `StaticFiles` where possible, as it securely handles path normalization and directory traversal protection.
