## 2026-06-23 - Path Traversal in FastAPI FileResponse SPA handler
**Vulnerability:** A path traversal vulnerability was found in the catch-all SPA route in `server/main.py`. The `serve_web_app` route used `dist_dir / full_path` and checked `.is_file()` before returning a `FileResponse`. This allowed traversing outside the base directory using dot segments `../` (e.g. `GET /../../../../etc/passwd` via raw TCP request).
**Learning:** In Python, `pathlib.Path(...).is_file()` evaluates to True as long as the resolved absolute path points to a valid file, meaning it does not inherently prevent path traversal. Additionally, FastAPI's `FileResponse` does not provide built-in directory boundary checks (unlike `StaticFiles`). Finally, standard HTTP clients (and FastAPI's `TestClient`) often normalize path dot segments locally before sending requests, making manual TCP or raw HTTP testing necessary to trigger the vulnerability.
**Prevention:** When manually serving static files or dynamic paths in FastAPI using `FileResponse` (like a catch-all SPA route), always use `target.resolve().is_relative_to(base_dir.resolve())` to explicitly enforce boundary checks and prevent Local File Inclusion (LFI). Ensure failed boundary checks gracefully fall back to the default `index.html` to maintain client-side routing.
