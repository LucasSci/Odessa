## 2026-06-01 - Prevent Path Traversal/LFI via FastAPI Catch-All Routes
**Vulnerability:** A catch-all route `@app.get("/{full_path:path}")` manually served files using `FileResponse` simply by appending the user input to the base path (`target = dist_dir / full_path`). This allowed critical Path Traversal / Local File Inclusion (LFI) via payload paths like `/../server/core/auth.py`.
**Learning:** `fastapi.responses.FileResponse` does not provide the built-in boundary checks that `StaticFiles` natively provides. Relying on simple path joining with string inputs exposes the entire file system relative to the mount point. Furthermore, FastAPI's `TestClient` normalizes `../` automatically locally before creating the mock request, completely obscuring the vulnerability from standard unit tests unless tested with raw HTTP or socket requests.
**Prevention:** When manually serving dynamic paths in FastAPI using `FileResponse`, always utilize `pathlib` features to perform boundary checks: `target.resolve().is_relative_to(base_dir.resolve())`. Test LFI using raw socket connections or real `httpx` HTTP requests against a live server instead of `TestClient`.
