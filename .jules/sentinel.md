## 2025-02-12 - Fix Overly Permissive CORS Configuration\n**Vulnerability:** CORS in `server/main.py` allows any origin ('*') while also setting `allow_credentials=True`, which is insecure and can expose the API to CSRF or data leaks in authenticated sessions.\n**Learning:** The memory hinted about it: "CORS origins in `server/main.py` are restricted using a whitelist defined in `server/config.py`... populated from the `CORS_ALLOWED_ORIGINS` environment variable". But this was not correctly implemented in `server/main.py`.\n**Prevention:** Load `CORS_ALLOWED_ORIGINS` in `server/config.py` as a string of comma-separated values, parse it into a list, and pass it to `CORSMiddleware` instead of using `["*"]`.
