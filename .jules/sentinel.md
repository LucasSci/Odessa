## 2026-05-25 - Prevent Path Traversal in Catch-All File Route
**Vulnerability:** The catch-all `serve_web_app` route (`/{full_path:path}`) in `server/main.py` directly concatenated the `full_path` string with a base directory and returned a `FileResponse` if it existed. By sending a raw HTTP request (bypassing normal client path normalization) like `GET /../server/main.py`, an attacker could traverse outside the intended `dist` directory and read arbitrary files on the system (LFI).
**Learning:** `FileResponse` in FastAPI does not perform the built-in directory boundary checks that `StaticFiles` provides. When manually serving files with a dynamic path segment, the developer must explicitly validate that the resolved target path remains relative to the intended base directory.
**Prevention:** Always use `path.resolve().is_relative_to(base_dir.resolve())` before passing user-supplied paths to `FileResponse`. Catch any exceptions during path resolution and fallback securely (e.g., serving `index.html`).
