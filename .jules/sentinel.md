## 2026-06-06 - Path Traversal (LFI) in FastAPI Static SPA Fallback
**Vulnerability:** The catch-all route `/{full_path:path}` in `server/main.py` was directly concatenating user-provided paths with the base directory and calling `target.is_file()` to return a `FileResponse`. This allowed trivial path traversal (e.g., `../server/main.py`) because `FileResponse` does not automatically validate paths like `StaticFiles` does.
**Learning:** When manually implementing an SPA fallback route in FastAPI using `FileResponse`, it is critical to explicitly enforce path boundaries using `pathlib`'s `.resolve()` and `.is_relative_to()`. The developer incorrectly assumed `is_file()` provided some security.
**Prevention:** Always use `.resolve().is_relative_to(base_dir.resolve())` when constructing file paths from user input. Do not rely solely on FastAPI `StaticFiles` semantics if manually serving files via `FileResponse`. Ensure failed checks fall back securely to the safe default (e.g., `index.html`) rather than returning a 500 error to maintain correct SPA routing behavior.
