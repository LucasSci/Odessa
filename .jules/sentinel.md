## 2026-06-02 - Path Traversal in FastAPI Catch-All Route
**Vulnerability:** A Local File Inclusion (LFI) / Path Traversal vulnerability in the catch-all route `@app.get("/{full_path:path}")` allowed reading arbitrary files (e.g. `/etc/passwd`) using `../` segments because `FileResponse` was supplied directly with a concatenated path without bounding checks.
**Learning:** In FastAPI, wildcard routes processing arbitrary strings without `StaticFiles` lack built-in boundaries. TestClient local testing can be deceptive as standard HTTP clients often normalize the URL string *before* sending, requiring raw HTTP requests or un-normalized tools (like `curl --path-as-is`) to discover the flaw.
**Prevention:** Always validate that manually constructed user-controlled paths resolve within a target directory using `target.resolve().is_relative_to(base_dir.resolve())` before passing them to `FileResponse`. Ensure fallback paths correctly serve the default Single Page Application HTML instead of triggering server errors if a path breaches boundaries.
