## 2026-06-09 - Path Traversal (LFI) in Catch-all Route
**Vulnerability:** A Local File Inclusion (LFI) vulnerability existed in `serve_web_app` (the FastAPI catch-all route for SPA) where user-provided paths `/{full_path:path}` were directly appended to the base directory and served via `FileResponse` without checking if the resolved path stayed within the base directory.
**Learning:** `FileResponse` does not provide the built-in boundary checks that `StaticFiles` does. When manually serving static files or dynamic paths in FastAPI, failing to check `path.resolve().is_relative_to(base_dir.resolve())` enables attackers to read arbitrary files (like the server's source code or config files) using `../` in the path.
**Prevention:** Always explicitly validate that a resolved user-provided path is relative to the resolved base directory before serving it with `FileResponse`, and bypass standard test clients to test this properly since `TestClient` normalizes `../` automatically before hitting the server.
