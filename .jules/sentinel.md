## 2024-05-18 - SSRF Vulnerability in Web Proxy
**Vulnerability:** The `/proxy` and `/proxy/asset` endpoints in `server/api/v1/endpoints/proxy.py` were vulnerable to Server-Side Request Forgery (SSRF).
**Learning:** `httpx` follows redirects by default (`follow_redirects=True`). When mitigating SSRF, simply checking the initial URL using a host resolution check isn't enough, as an attacker could provide a public URL that subsequently redirects to an internal one, bypassing the initial check.
**Prevention:** Implement an `_is_safe_url` helper to resolve and check IPs against private/loopback address spaces. Set `follow_redirects=False` in the HTTP client to manually handle redirects. Ensure you recursively validate every redirect URL against the `_is_safe_url` check. Add a strict `redirect_count` limit to prevent Denial of Service (DoS) attacks via infinite redirect loops.
