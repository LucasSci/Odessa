## 2026-06-24 - LFI in Catch-All Routing
**Vulnerability:** A Local File Inclusion (LFI) vulnerability existed in `server/main.py` where a catch-all route `/{full_path:path}` used `FileResponse(dist_dir / full_path)` to serve Single Page Application (SPA) static files without proper boundary checks.
**Learning:** `pathlib.Path(...).is_file()` alone is insufficient to prevent path traversal because it evaluates to True as long as the resolved absolute path points to a valid file, meaning it does not inherently prevent path traversal when evaluating user-provided paths without an explicit boundary check like `is_relative_to()`. The file check must be coupled with a path boundary validation.
**Prevention:** Always use `target.resolve().is_relative_to(base_dir.resolve())` to ensure the final path remains within the intended base directory. Avoid directly passing unfiltered user paths to `FileResponse` inside catch-all routes; ensure failed validations gracefully fallback to default safe routes like `index.html`.
