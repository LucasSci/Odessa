## 2026-06-19 - Fast API Catch-All Route Path Traversal
**Vulnerability:** A catch-all route (`/{full_path:path}`) returning `FileResponse` was only checking `target.is_file()` to verify if a requested file exists, which didn't prevent path traversal vulnerabilities (`../`) allowing attackers to read files outside the designated `dist` folder.
**Learning:** `FastAPI`'s `FileResponse` doesn't enforce path traversal boundary checks natively the same way `StaticFiles` does. `pathlib.Path.is_file()` will return true as long as the resolved target is a real file, regardless of whether it was resolved via traversal or directly.
**Prevention:** Whenever serving dynamic paths manually (like using a catch-all SPA route with `FileResponse`), the target must be securely resolved and bounded using `target.resolve().is_relative_to(base_dir.resolve())` before doing any further verification or handling. Make sure both paths are explicitly `.resolve()`'d otherwise the evaluation will be `False` and legitimate SPA routing could break.
