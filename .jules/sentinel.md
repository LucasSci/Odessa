## 2026-05-24 - Path Traversal in FastAPI FileResponse Catch-All Route
**Vulnerability:** A catch-all route `/{full_path:path}` in `server/main.py` was directly appending user input to a base directory (`target = dist_dir / full_path`) and returning it via `FileResponse(target)`. This allowed attackers to use `../` payloads (e.g., `/../server/main.py`) to access any file readable by the server process outside the intended `dist` directory.
**Learning:** While FastAPI's `StaticFiles` component automatically handles directory boundary checks, manually serving files using `FileResponse` combined with a catch-all route does NOT. Furthermore, standard HTTP testing clients (like FastAPI's `TestClient` or `httpx`) often normalize paths (removing `../`) before sending the request, hiding the vulnerability during normal integration testing.
**Prevention:** Always use `path.resolve().is_relative_to(base_dir.resolve())` to enforce strict boundary checks when serving files dynamically with `FileResponse`. To properly test for path traversal, use raw socket connections or specialized tools that do not automatically normalize the HTTP request path.
