## 2026-06-03 - [FastAPI FileResponse Path Traversal in SPA Catch-all Routes]
**Vulnerability:** A catch-all route `@app.get("/{full_path:path}")` used to serve a React SPA with `FileResponse` was vulnerable to Local File Inclusion (LFI). The path input was directly appended to a base directory (`dist_dir / full_path`) without checking if the resolved path stayed within the base directory.
**Learning:** `FileResponse` does not provide built-in boundary checks to prevent path traversal unlike `StaticFiles`. Even with `../` normalization by browsers/some clients, tools like `curl --path-as-is` or raw HTTP requests bypass this, allowing reading of arbitrary files on the system like `/etc/passwd`.
**Prevention:** When manually serving dynamic paths with `FileResponse`, always resolve the path first (`(base_dir / path).resolve()`) and explicitly verify boundaries using `target.is_relative_to(base_dir)`. If the check fails in an SPA catch-all context, it should gracefully fall back to the default `index.html` rather than returning a 403 or 500 error, to preserve correct client-side routing behavior.
