## 2025-02-12 - Prevent O(N) memory allocation with reverse array searching
**Learning:** In the `OdessaLiveCenter.tsx` file, continuously growing arrays (like `capturedText`) were being queried using `[...capturedText].reverse().find(...)`. This is a severe anti-pattern for performance because the spread operator `[...]` allocates a brand new array matching the length of the source, and `.reverse()` creates another copy or mutates it before `.find()` iterates. In React, doing this inside a hook or render loop forces O(N) memory allocation and processing overhead on every render, which will quickly bottleneck as the chat log grows. Since the project uses an older target (ES2020), native zero-copy methods like `findLast()` are unavailable.
**Action:** Replace `[...array].reverse().find(...)` with a traditional backward `for` loop (e.g., `for (let i = array.length - 1; i >= 0; i--)`). This avoids the memory allocation entirely, operating with O(1) space overhead and cleanly short-circuiting once the item is found.
