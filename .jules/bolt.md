## 2024-05-23 - Prevent Render Bottlenecks in React Components Array Filtering
**Learning:** React components (e.g., `CaptureStudio`, `OdessaLiveCenter`, `ContentLibrary`) with high-frequency state updates derive multiple counts or subsets from continuously growing arrays (like event logs or content lists). Using multiple chained `.filter()` calls inside render or `useMemo` created O(N) operations for each specific check (e.g. `items.filter(x => x.enabled)`, `items.filter(x => x.safety)`), which significantly slowed down rendering performance and caused frame drops.
**Action:** Always combine derived computations on large or continuously growing arrays into a single-pass loop or `reduce` inside a `useMemo` hook, rather than chaining multiple array iterations, to optimize rendering performance in React components with heavy event streams.
