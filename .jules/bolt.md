## 2024-03-24 - [Avoid unnecessary map/filter recalculations for array-derived state during frequent rendering]
**Learning:** Found that `CaptureStudio.tsx` recalculates `successfulEvents`, `averageConfidence`, and `averageLatency` by re-mapping and reducing the `captureEvents` array on every single render. Given how often UI interactions happen during live capture and stream configuration, this scales poorly with the event array size.
**Action:** Used `useMemo` to cache arrays derived from other array state (like filtering active/successful elements) and wrapped `CaptureStudio` component with `React.memo` to improve rendering performance.

## 2024-05-13 - [Combine array operations into single-pass loops for continuously growing React state]
**Learning:** In `src/core/useAutopilotRuntime.ts`, deriving values like `completedCycles`, `failedCycles`, and `averageConfidence` using multiple `.filter()`, `.map()`, and `.reduce()` calls on the growing `cycles` array caused significant performance degradation due to multiple O(N) operations per render. This is particularly problematic in live systems where state updates frequently.
**Action:** When computing multiple derived metrics from a continuously growing array in high-frequency React state hooks, combine the operations into a single-pass `for` loop or `.reduce()` inside a single `useMemo` block. This reduces iterations and avoids unnecessary garbage collection of intermediate arrays.
