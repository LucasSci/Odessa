## 2026-06-19 - Array Reverse Find Performance Anti-pattern\n**Learning:** React components (like `OdessaLiveCenter`) frequently processing continuously growing arrays (like event logs or OCR capture text) will suffer severe render bottlenecks when searching backwards using `[...array].reverse().find(...)`. This forces O(N) memory allocation to clone the array, O(N) time to reverse it, and then iterates, all on the main UI thread during every single re-render. Since `findLast` is not available natively in ES2020 TS targets, this anti-pattern is common.\n**Action:** Always replace `[...array].reverse().find(...)` with a simple backward `for` loop to achieve O(1) memory and immediately break when the target is found. This applies to any large, continuously growing array in React render paths.
