Fluxograma da lógica do Odessa

Mapa visual do que é executado quando a live gera texto ou eventos: da tela de captura, passando pelo OCR e pela ingestão, até gatilhos, fila, vídeos, OBS, TTS, webhooks e logs.

Leitura rápida

  • Rota principal: janela/OBS -> OCR -> ingestão -> parser -> gatilho -> executor.
  • Rota paralela: Event Bus local alimenta Autopilot/Persona Runtime.
  • Ajuste crítico: qualidade OCR + classificação por zona + gatilhos salvos.
  • Risco atual: TTS desabilitado e teste de cena com payload divergente.

Pendência

TTS / cena

Testes indicam TTS desabilitado com path vazio e payload de cena divergente.

Contrato essencial: o frontend deve sempre enviar texto + contexto (source, zoneName, kind, metadata).
Melhor lugar para diagnosticar: compare texto bruto OCR, evento parseado, matchedTriggers, queuedActions, executions e videoState.
Rotas montadas: backend expõe /api/v1/*, /api/* e algumas rotas raiz de compatibilidade para OBS/OCR/Webhooks.

Legenda

Captura real: caminho janela/OBS -> OCR -> ingestão -> ação.
Paralelo: testes manuais e Event Bus/Autopilot.
Configuração: edição de gatilhos, vídeos e refresh do engine.
Ajuste: pontos onde vale mexer para aumentar captura efetiva.
Pendência: falhas vistas em testes completos, fora da captura principal.

Rotas-chave

  • /healthStatus geral do backend, OCR, IA e TTS.
  • /api/v1/ocr/processOCR de uma zona já recortada pela tela.
  • /api/v1/ocr/obs-cycleOCR de múltiplas zonas a partir da source OBS.
  • /api/v1/automation/ingestEntrada única para texto OCR/manual virar evento e ação.
  • /api/v1/automation/logsRastro de parser, trigger, fila e executor.
  • /api/v1/video/stateEstado atual que palco/overlay devem refletir.
  • /obs/switch-sceneSaída para troca de cena OBS.

Checklist de ajuste

  • 1. A tela detecta frames?Verificar preview, frameHash e aviso de frame repetido.
  • 2. O OCR vê texto bruto?Conferir painel "Texto bruto" por zona.
  • 3. O evento foi classificado certo?Conferir summary.events em /automation/logs.
  • 4. Algum trigger casou?Conferir matchedTriggers e cooldown.
  • 5. A ação executou?Conferir executions, videoState, OBS, TTS ou webhook.
